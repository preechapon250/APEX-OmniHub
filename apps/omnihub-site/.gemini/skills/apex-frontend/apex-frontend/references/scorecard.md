# APEX Frontend Scorecard (target ≥ 90/100)

## Scoring
1) UX clarity & task success (0–25)
- Clear job story, flow matches user intent, friction removed

2) UI craft & consistency (0–20)
- Tokens used consistently, spacing/type intentional, cohesive aesthetics

3) State completeness (0–10)
- loading/empty/error/partial/success/offline/permission handled

4) Accessibility (0–15)
- labels, focus order, contrast, targets, reduced motion

5) Performance awareness (0–15)
- budgets defined; hot path addressed; avoids common traps

6) Engineering quality (0–15)
- architecture separation, tests, maintainability, clear contracts

## Auto-fail
- Missing error/empty states on critical screen
- Inaccessible primary actions (no labels / unreachable)
- “Fix” without a repro or without prevention
