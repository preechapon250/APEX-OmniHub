# DESIGN Protocol (Universal)

## Step 0: Contract
- Users + job stories
- Platform/stack
- Primary metric
- Constraints (brand/a11y/perf)
- Risks

## Step 1: Flow
- Map 3â€“7 steps as verbs
- Define navigation model and escape hatches

## Step 2: Wireframes
- Thumb-first (mobile)
- Keyboard-first (web/desktop)
- Always design: loading/empty/error/partial/success/offline/permission

## Step 3: UI Spec
- Tokens: color roles, type scale, spacing, radius, motion
- Component inventory
- Micro-interactions

## Step 4: Microcopy
- CTA formula: Verb + object + outcome
- Error formula: what + why + fix + preserve input

## Output format (required)
- Flow map
- Wireframes
- State table
- Component inventory
- Acceptance criteria
- A11y checklist
